Troubleshooting Guide

Common security issues and their solutions. Learn from mistakes I've encountered during competitions and how to fix them quickly.

Common Security Mistakes

These are frequent errors that can cost you points in competitions or leave your system vulnerable.

BitLocker Not Starting After Enabling

Problem

BitLocker wizard completes but encryption never begins, or status shows "Waiting for Activation"

Solutions

  1. Verify TPM is enabled and initialized
    PowerShell (Admin)
    Get-Tpm
# Check that TpmPresent and TpmReady are both True
  2. Ensure sufficient free space

    BitLocker requires at least 5% free space on the drive being encrypted

  3. Check if system partition meets requirements

    System drive must have a separate system partition (usually 100-500 MB)

  4. Resume encryption manually
    PowerShell (Admin)
    Resume-BitLocker -MountPoint "C:"

Password Policy Not Applied to Existing Users

Problem

Changed password policy settings but existing users still have weak passwords

Solution

Password policies only affect new passwords or password changes. Force users to change passwords:

PowerShell (Admin)
# Force single user to change password at next login
Set-LocalUser -Name "username" -PasswordNeverExpires $false

# Force all users except yourself
Get-LocalUser | Where-Object {$_.Name -ne "YourUsername"} | Set-LocalUser -PasswordNeverExpires $false

Important: Change your own password first to meet requirements before forcing others!

Audit Logs Not Recording Events

Problem

Enabled audit policies but no events appear in Security log

Solutions

  1. Verify audit policy is actually enabled
    Command Prompt (Admin)
    auditpol /get /category:*
  2. Check Security log size and retention

    Open Event Viewer → Right-click Security log → Properties

    Ensure "Maximum log size" is at least 100 MB and retention is configured

  3. Force a test event

    Try logging in with wrong password to generate Event ID 4625

  4. Restart after policy changes

    Some audit policies require restart to take effect

Firewall Blocking Legitimate Applications

Problem

After enabling firewall, certain applications or services stop working

Solutions

  1. Check firewall logs

    Enable firewall logging: Firewall Advanced Settings → Right-click profile → Properties → Logging

  2. Create application exception

    Windows Firewall → Allow an app → Add the blocked application

  3. Check for conflicting rules

    Review both inbound and outbound rules for conflicts or duplicates

  4. Test with firewall temporarily disabled

    Disable firewall briefly to confirm it's the cause, then re-enable and create proper exception

Cannot Disable Guest Account

Problem

Guest account keeps getting re-enabled or shows as enabled despite disabling it

Solutions

  1. Disable via PowerShell
    PowerShell (Admin)
    Disable-LocalUser -Name "Guest"
Get-LocalUser -Name "Guest" | Select Name, Enabled
  2. Disable via Local Security Policy

    secpol.msc → Local Policies → Security Options → Accounts: Guest account status → Disabled

  3. Check for Group Policy Override

    Run gpresult /r to check if domain policy is re-enabling it

Account Lockout Too Aggressive

Problem

Users getting locked out too frequently, even with legitimate login attempts

Solutions

  1. Adjust lockout threshold

    5 attempts is recommended balance between security and usability

  2. Set reasonable lockout duration

    30 minutes allows recovery without admin intervention

  3. Check for password sync issues

    Saved credentials in browsers or apps may be trying old passwords

  4. Review Event ID 4740

    Identify which machine/process is causing failed attempts

Windows Update Issues

Windows Update Stuck or Failing

Solutions

  1. Run Windows Update Troubleshooter

    Settings → Update & Security → Troubleshoot → Windows Update

  2. Reset Windows Update components
    Command Prompt (Admin)
    net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
ren C:\Windows\SoftwareDistribution SoftwareDistribution.old
ren C:\Windows\System32\catroot2 catroot2.old
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
  3. Check disk space

    Ensure C: drive has at least 10 GB free space

  4. Manually download and install updates

    Visit Microsoft Update Catalog if automatic updates fail

Network & Connectivity Issues

Network Connection Lost After Security Changes

Solutions

  1. Check firewall rules

    Ensure DHCP Client (UDP 67, 68) and DNS (TCP/UDP 53) are allowed

  2. Verify network services are running
    PowerShell (Admin)
    Get-Service -Name Dhcp, Dnscache, LanmanWorkstation | Select Name, Status
  3. Check if network adapter is disabled

    Device Manager → Network adapters → Ensure adapter is enabled

  4. Reset TCP/IP stack
    Command Prompt (Admin)
    netsh int ip reset
netsh winsock reset
ipconfig /flushdns
ipconfig /renew

Performance Issues After Hardening

System Running Slow After Security Configuration

Common Causes & Solutions

  1. BitLocker encryption in progress

    Check encryption status - system will be slower until encryption completes

    PowerShell
    Get-BitLockerVolume | Select MountPoint, EncryptionPercentage
  2. Windows Defender full scan running

    Schedule scans during off-hours: Windows Security → Virus & threat protection → Scan options

  3. Excessive audit logging

    Reduce logging to only essential categories if performance is critical

  4. Check for malware

    Run full antivirus scan - sometimes slowness indicates infection

Quick Diagnostic Commands

Essential commands for troubleshooting security configurations

Check Security Status

PowerShell
# Get security center status
Get-MpComputerStatus

# Check firewall status
Get-NetFirewallProfile | Select Name, Enabled

# BitLocker status
Get-BitLockerVolume

# TPM status
Get-Tpm

User Account Information

PowerShell
# List all local users
Get-LocalUser | Select Name, Enabled, PasswordRequired

# Check user group membership
Get-LocalGroupMember -Group "Administrators"

# View locked accounts
Get-LocalUser | Where-Object {$_.Locked -eq $true}

Audit Policy Status

Command Prompt
# View all audit policies
auditpol /get /category:*

# Check specific category
auditpol /get /category:"Logon/Logoff"

System & Service Info

PowerShell
# Running services
Get-Service | Where-Object {$_.Status -eq "Running"}

# Startup programs
Get-CimInstance Win32_StartupCommand

# Installed software
Get-ItemProperty HKLM:\Software\Microsoft\Windows\CurrentVersion\Uninstall\*

Competition-Specific Tips

Time Management

  • Start with quick wins: disable Guest, enable firewall, basic password policy
  • Don't spend too much time on one issue - move on and come back later
  • Keep track of what you've already done using a checklist
  • Test each change to ensure it doesn't break functionality

Common Pitfalls to Avoid

  • Don't lock yourself out by changing your own password incorrectly
  • Don't disable services required for scoring (varies by scenario)
  • Don't forget to actually start BitLocker encryption (not just set it up)
  • Don't skip README files - they contain critical information
  • Don't assume default settings are secure - verify everything
