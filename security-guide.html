BitLocker Drive Encryption

Protect your data from physical theft and unauthorized access

What is BitLocker?

BitLocker is a full-disk encryption feature included with Windows that protects your data by encrypting entire drives. Even if someone physically steals your hard drive, they cannot access your data without the encryption key.

Why Use BitLocker?

  • Protects against data theft from lost or stolen devices
  • Prevents unauthorized access to sensitive files
  • Required for many security compliance standards
  • Transparent to users once configured

Requirements

Windows Editions: Windows 10/11 Pro, Enterprise, or Education

Hardware: TPM (Trusted Platform Module) 1.2 or higher (recommended)

Step-by-Step Guide

Step 1

Check TPM Status

PowerShell (Admin) 
Get-Tpm

Verify that TPM is present and ready. Look for "TpmPresent: True" and "TpmReady: True"

Step 2

Open BitLocker Settings

Navigate to Control Panel → System and Security → BitLocker Drive Encryption

Or search for "BitLocker" in Windows Search

Step 3

Enable BitLocker

  1. Click "Turn on BitLocker" next to your system drive (usually C:)
  2. Wait for the initialization to complete
  3. Choose how to unlock your drive at startup (TPM recommended)
  4. Save your recovery key in a safe location (critical!)
  5. Choose encryption mode (encrypt used space only for faster setup)
  6. Run the BitLocker system check
  7. Restart your computer to begin encryption
Step 4

Verify Encryption

PowerShell (Admin) 
Get-BitLockerVolume

Check that "ProtectionStatus" shows "On" and "EncryptionPercentage" shows 100

Critical: Save Your Recovery Key!

Store your recovery key in multiple secure locations. If you lose access to your account and don't have the recovery key, your data will be permanently inaccessible. Print it, save it to a USB drive (stored separately), or save it to your Microsoft account.

Audit Logging & Monitoring

Monitor and track security-relevant events on your system

What are Audit Logs?

Audit logs record security-related events on your system, such as login attempts, file access, permission changes, and system modifications. They are essential for detecting unauthorized access and investigating security incidents.

Critical Events to Monitor

  • Failed login attempts (potential brute force attacks)
  • Account lockouts and password changes
  • Privilege escalation attempts
  • Changes to audit policies
  • System startup and shutdown
  • Installation of new software or services

Enable Advanced Audit Policies (Windows)

Step 1

Open Local Security Policy

Press Win + R, type secpol.msc, and press Enter

Step 2

Navigate to Audit Policies

Security Settings → Advanced Audit Policy Configuration → Audit Policies

Step 3

Enable Key Audit Categories

For each category below, double-click and enable both Success and Failure auditing:

  • Account Logon: Audit Credential Validation
  • Account Management: Audit User Account Management, Audit Security Group Management
  • Logon/Logoff: Audit Logon, Audit Logoff, Audit Account Lockout
  • Policy Change: Audit Audit Policy Change
  • Privilege Use: Audit Sensitive Privilege Use
  • System: Audit Security State Change, Audit System Integrity
Step 4

View Audit Logs

Open Event Viewer (Win + R → eventvwr.msc)

Navigate to Windows Logs → Security

Common Event IDs to Watch

Event ID Description Severity
4624 Successful account logon Info
4625 Failed logon attempt Warning
4648 Logon using explicit credentials Info
4720 User account created Warning
4726 User account deleted Critical
4732 Member added to security group Warning
4740 User account locked out Critical

Pro Tip: Regular Log Review

Review your security logs weekly. Look for patterns of failed logins, unusual account activity, or unexpected privilege changes. Set up log retention for at least 90 days.

Password Policy Configuration

Enforce strong password requirements across your system

Why Password Policies Matter

Weak passwords are one of the most common security vulnerabilities. A strong password policy ensures that all user accounts have passwords that resist brute-force attacks and dictionary attacks.

Recommended Password Requirements

  • Minimum length: 12 characters (14+ for administrator accounts)
  • Must contain uppercase and lowercase letters
  • Must contain numbers
  • Must contain special characters (!@#$%^&*)
  • Cannot contain username or common words
  • Password history: Remember last 24 passwords
  • Maximum password age: 90 days
  • Minimum password age: 1 day
  • Account lockout after 5 failed attempts

Configure Password Policy (Windows)

Step 1

Open Local Security Policy

Run Command 
secpol.msc
Step 2

Navigate to Password Policy

Security Settings → Account Policies → Password Policy

Step 3

Configure Each Setting

Enforce password history 24 passwords remembered
Maximum password age 90 days
Minimum password age 1 day
Minimum password length 12 characters
Password must meet complexity requirements Enabled
Store passwords using reversible encryption Disabled
Step 4

Configure Account Lockout Policy

Navigate to Account Policies → Account Lockout Policy

Account lockout threshold 5 invalid attempts
Account lockout duration 30 minutes
Reset lockout counter after 30 minutes

Creating Strong Passwords

Weak: password123, qwerty, admin2024

Too short, common words, predictable patterns

Strong: Tr0pic@lSt0rm#2026!, B1u3M00n$R15ing

12+ characters, mixed case, numbers, special characters

Passphrase: Correct-Horse-Battery-Staple-99!

Long, memorable, random words with modifications

Never Reuse Passwords

Use unique passwords for every account. If one service is compromised, attackers will try those credentials on other sites. Consider using a password manager to store unique passwords securely.

Firewall Configuration

Control network traffic and protect against external threats

What is a Firewall?

A firewall monitors and controls incoming and outgoing network traffic based on predetermined security rules. It acts as a barrier between your trusted internal network and untrusted external networks.

Windows Firewall Configuration

Step 1

Open Windows Defender Firewall

Control Panel → System and Security → Windows Defender Firewall

Or search for "Firewall" in Windows Search

Step 2

Enable Firewall for All Profiles

Click "Turn Windows Defender Firewall on or off" and ensure firewall is ON for:

  • Domain networks
  • Private networks
  • Public networks (most important!)
Step 3

Configure Advanced Settings

Click "Advanced settings" to access Windows Defender Firewall with Advanced Security

Step 4

Review Default Rules

Review Inbound Rules and Outbound Rules. Disable or remove any unnecessary rules for services you don't use.

Creating Custom Firewall Rules

Example: Block a specific program from accessing the internet

  1. Open Windows Defender Firewall with Advanced Security
  2. Right-click "Outbound Rules" → "New Rule"
  3. Select "Program" → Click "Next"
  4. Browse to the program's executable file
  5. Select "Block the connection"
  6. Apply to all profiles (Domain, Private, Public)
  7. Name the rule and add a description

Best Practices

  • Enable firewall on all network profiles
  • Block all inbound connections by default (whitelist approach)
  • Only allow necessary outbound connections
  • Disable unused network services and protocols
  • Regularly review firewall logs for suspicious activity
  • Use separate rules for different network zones

Testing Your Firewall

Use online tools like ShieldsUP! (grc.com) to test your firewall configuration. A properly configured firewall should show all ports as "stealth" (appearing as if the computer doesn't exist).

User Account Management

Control who has access to your system and what they can do

Principle of Least Privilege

Users should only have the minimum permissions necessary to perform their tasks. This limits the damage that can be done if an account is compromised.

Disable Unnecessary Accounts

Step 1

Review All User Accounts

PowerShell (Admin) 
Get-LocalUser
Step 2

Disable Guest Account

PowerShell (Admin) 
Disable-LocalUser -Name "Guest"
Step 3

Remove Unauthorized Users

Delete or disable any accounts that are:

  • No longer needed
  • Created by malware
  • Unknown or unauthorized
  • Test or temporary accounts

Secure Administrator Accounts

  • Rename the default Administrator account
  • Use a separate admin account for administrative tasks
  • Use a standard user account for daily activities
  • Enable User Account Control (UAC)
  • Require strong passwords for all admin accounts
  • Limit the number of administrators

Group Management

Properly assign users to security groups:

Group Purpose Recommendation
Administrators Full system control Minimize membership, only IT staff
Power Users Limited admin rights Legacy group, avoid if possible
Users Standard privileges Default for all regular users
Guests Minimal access Disable guest account entirely

Don't Use Admin for Daily Tasks

Using an administrator account for everyday activities (web browsing, email, etc.) puts your system at greater risk. Malware running with admin privileges can cause significantly more damage. Always use a standard user account for daily tasks.
